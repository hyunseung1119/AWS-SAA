AWS란?

클라우드 인프라를 직접 조립하지 않고, 서비스 형태로 쓰는 방식

AWS는 서버, 데이터베이스, 네트워크, 보안 기능을 필요할 때 바로 사용하는 클라우드 플랫폼입니다. AWS SAA는 "요구사항(가용성, 성능, 보안, 비용)에 맞는 아키텍처를 설계할 수 있는지"를 검증합니다.

왜 필요한가

서비스 확장, 장애 복구, 보안 설계를 표준화하고 빠르게 구현할 수 있습니다.

SAA 핵심

단일 서비스 암기가 아니라 "트레이드오프 판단"이 핵심입니다.

학습 방식

개념 → 용어 → 시나리오 문제 → 꼬리질문 순으로 반복 학습합니다.

시험 포인트

SAA-C03 도메인별 관점

보안 설계

IAM 최소 권한, 네트워크 경계, 데이터 암호화, 비밀정보 관리

회복탄력성

Multi-AZ, 백업, 장애조치, 느슨한 결합, 재시도/큐잉 전략

고성능 설계

캐시, CDN, 오토스케일링, 데이터 액세스 패턴 최적화

비용 최적화

요금 모델 선택, 데이터 수명주기, 관리형 서비스 우선
